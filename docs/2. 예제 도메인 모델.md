# 2. 예제 도메인 모델

## 예제 도메인 모델과 동작확인

### 엔티티 클래스
* **Member**
    * `id`: `Long`
    * `username`: `String`
    * `age`: `int`
    * `team`: `Team`
* **Team**
    * `id`: `Long`
    * `name`: `String`
    * `members`: `List<Member>`

### ERD
* `member` 테이블과 `team` 테이블이 있으며, `member.team_id`가 `team.team_id`를 참조하는 외래 키 관계입니다.

### 롬복 설명
* **`@Setter`**: 실무에서 가급적 Setter는 사용하지 않는 것이 좋습니다.
* **`@NoArgsConstructor(access = AccessLevel.PROTECTED)`**: 기본 생성자를 막고 싶은데, JPA 스펙상 `PROTECTED`로 열어두어야 합니다.
* **`@ToString`**: 가급적 내부 필드만(연관관계 없는 필드만) 포함하는 것이 좋습니다.
* **`changeTeam()`**: 양방향 연관관계를 한번에 처리하기 위한 연관관계 편의 메소드입니다.

### 연관관계
* `Member`와 `Team`은 양방향 연관관계입니다.
* `Member.team`이 연관관계의 주인입니다.
* `Team.members`는 연관관계의 주인이 아니므로, `Member.team`이 데이터베이스 외래키 값을 변경하고, 반대편은 읽기만 가능합니다.
